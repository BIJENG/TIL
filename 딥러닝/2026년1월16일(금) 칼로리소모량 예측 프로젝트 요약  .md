# 📊 칼로리 소모량 예측 AI 프로젝트 (Master Ver.)

## 1. 프로젝트 배경 및 목적
본 프로젝트는 **운동 데이터(신체 정보 + 운동 지표)**를 기반으로 개인의 칼로리 소모량을 예측하는 회귀 모델 구축을 목표로 했습니다. 단순한 정확도 경쟁을 넘어, **헬스케어 데이터의 특성(이상치, 해석 가능성)을 고려한 실무형 파이프라인**을 설계하는 데 중점을 두었습니다.

* **타깃 변수**: Calories_Burned (정수형)
* **핵심 난제**: 정수형 타깃 분포 특성, 과적합(Overfitting) 방지, 모델의 설명력(Explainability) 확보

## 2. 데이터 분석 및 전처리 (EDA & Preprocessing) [보완]
### 2-1. 도메인 지식 기반 파생 변수 생성
단순히 주어진 변수만 사용하지 않고, 생리학적 지식을 활용해 모델의 예측력을 높였습니다.
* **BMI (체질량지수)**: `Weight / (Height/100)^2` 공식을 사용하여 비만도 관련 파생 변수 추가.
* **운동 강도 지수**: `Heart_Rate × Duration`을 통해 단순 시간이 아닌 '총 운동 부하'를 수치화.

### 2-2. 이상치 탐지 (Physiological Validation)
의료 데이터 특성상 기기 오류로 인한 비정상 수치가 존재할 가능성을 배제하지 않았습니다.
* Boxplot 및 IQR 방식을 사용하여 **생리학적으로 불가능한 수치**(예: 심박수 > 220, 체온 > 42도)를 사전에 필터링하여 데이터 품질을 확보했습니다.

## 3. 시행착오 및 문제 해결 (Troubleshooting)

### 🛑 문제 1: 과적합과 모델 불안정성
* **상황**: Gradient Boosting 등 복잡한 모델 사용 시 Validation 점수는 높으나 Test 점수가 급락하는 현상 발생.
* **원인**: 데이터 규모 대비 모델이 너무 복잡하여 훈련 데이터의 노이즈까지 학습함.
* **해결**: **"Occam's Razor(오컴의 면도날)"** 원칙 적용. 해석이 가능하고 일반화 성능이 뛰어난 **선형 회귀(Linear Regression) 및 릿지(Ridge) 모델**로 회귀.

### 🛑 문제 2: 정수형 타깃의 특성 간과
* **상황**: 회귀 모델은 소수점 예측값을 내놓지만, 실제 정답은 정수(Integer)임. 미세한 소수점 오차가 누적되어 성능 저하.
* **해결**: **후처리(Post-processing) 라운딩 기법** 도입. 예측값을 가장 가까운 정수로 반올림(`Round`)하여 제출, 리더보드 점수 대폭 개선.

## 4. 최종 모델 파이프라인
1.  **전처리**: 결측치 제어 및 StandardScaler 적용 (선형 모델은 스케일에 민감하므로 필수)
2.  **특성 공학**: BMI, 운동 강도 등 도메인 변수 추가
3.  **모델링**: Ridge Regression (L2 규제로 과적합 제어)
4.  **후처리**: 예측값 Rounding 처리

## 5. 핵심 인사이트
* **"Simple is Best"**: 무조건 최신 딥러닝 모델이 답은 아니다. 데이터 양과 분포에 맞는 모델 선정이 우선이다.
* **도메인 지식의 힘**: 알고리즘 튜닝보다 의미 있는 변수(BMI 등) 하나가 성능 향상에 더 기여한다.
* **해석 가능성(Explainability)**: 헬스케어 분야에서는 "왜(Why)" 그런 예측이 나왔는지 설명할 수 있는 선형 모델의 가치가 크다.
