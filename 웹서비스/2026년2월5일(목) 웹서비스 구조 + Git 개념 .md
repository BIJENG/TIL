# 📝 TIL: 웹 서비스 구조 심화 & Git 버전 관리 기초

## 1. 웹 서비스의 구성 요소 (Web Service Structure)

### 🖥️ 프론트엔드 (Frontend)
사용자가 직접 보고 조작하는 화면과 그 구성 요소를 담당하는 영역입니다.

* **역할**:
    * 사용자의 입력(클릭, 타이핑 등)을 받습니다.
    * 사용자의 행동을 **HTTP 요청(Request)**으로 변환하여 서버에 보냅니다.
    * 서버로부터 받은 응답(Response)을 해석하여 화면에 보여줍니다.
* **기술 스택**: HTML(구조), CSS(스타일), JavaScript(동작), React/Vue 등 프레임워크.
* **클라이언트 vs 프론트엔드**:
    * **클라이언트**: 사용자 측에서 실행되는 모든 프로그램(앱, 웹 브라우저 등).
    * **프론트엔드**: 웹 브라우저에서 실행되는 UI 및 상호작용 영역.

### ⚙️ 백엔드 (Backend)
눈에 보이지 않는 서버 뒤편에서 데이터 처리와 비즈니스 로직을 담당합니다.

* **역할**:
    * 클라이언트의 요청을 해석하고 적절한 로직을 수행합니다.
    * **데이터베이스(DB)**에 접속하여 데이터를 조회, 저장, 수정합니다.
    * 처리 결과를 HTTP 응답으로 만들어 반환합니다.
    * 보안, 인증, API 설계를 책임집니다.
* **기술 스택**: Python(Django/FastAPI), Java(Spring), Node.js 등.
* **서버 vs 백엔드**:
    * **서버**: 프로그램이 실행되는 컴퓨터(하드웨어/인프라).
    * **백엔드**: 서버 위에서 돌아가는 소프트웨어(로직).

## 2. 데이터와 저장소 (Data & Database)

데이터를 단순히 쌓는 것이 아니라, 효율적으로 관리하고 활용하기 위해 데이터베이스를 사용합니다.

### 🗄️ 데이터베이스 (Database)
* **필요성**: 체계적 저장, 빠른 검색, 데이터 일관성 유지, 다수 사용자의 동시 접근 지원.
* **RDBMS (관계형 데이터베이스)**: 데이터를 **표(Table)** 형태로 저장하고 관리하는 시스템.
    * **Table**: 행(Row)과 열(Column)로 구성된 데이터 집합.
    * **Primary Key (기본키)**: 각 행을 구분하는 고유 식별자 (예: ID).
    * **Foreign Key (외래키)**: 다른 테이블과 연결하기 위한 키 (예: 주문 내역의 user_id).

### ⌨️ SQL (Structured Query Language)
데이터베이스와 소통하기 위한 표준 언어입니다. (CRUD)
* **생성 (Create)**: `INSERT INTO users (name) VALUES ('alex');`
* **조회 (Read)**: `SELECT * FROM users;`
* **수정 (Update)**: `UPDATE users SET name='bob' WHERE id=1;`
* **삭제 (Delete)**: `DELETE FROM users WHERE id=1;`

## 3. 네트워크 기초 (Network Basics)

컴퓨터들이 서로 데이터를 주고받기 위한 연결 구조입니다.

### 📮 주소 체계 (IP & Port)
데이터를 정확한 곳으로 보내기 위해 두 가지 정보가 필수입니다.
* **IP 주소 (Address)**: 네트워크 상의 **컴퓨터(건물)** 위치. (예: `142.250.206.46`)
* **Port 번호 (Door)**: 컴퓨터 안의 **특정 프로그램(호수)**. (예: `80`, `443`, `3306`)
    * `80`: HTTP (일반 웹)
    * `443`: HTTPS (보안 웹)

### 🌐 DNS (Domain Name System)
* **역할**: 외우기 힘든 IP 주소(숫자) 대신, 기억하기 쉬운 **도메인 이름(문자)**을 사용하게 해주는 전화번호부 시스템.
* **과정**: 사용자가 `google.com` 입력 -> DNS가 IP(`142....`)로 변환 -> 해당 IP로 접속.

## 4. Git 기초 (Git Basics)

### 🌲 버전 관리 (Version Control)
* **필요성**:
    * 실수했을 때 과거 시점으로 복구 가능.
    * "최종.txt", "진짜최종.txt" 같은 파일 복사 지옥 탈출.
    * 여러 사람이 동시에 작업할 때 충돌 방지 및 협업 용이.
* **Git vs GitHub**:
    * **Git**: 내 컴퓨터에서 버전 관리를 하는 **도구(프로그램)**.
    * **GitHub**: Git으로 관리한 코드를 업로드하는 **클라우드 저장소(서비스)**.

### 🗝️ 핵심 용어
1.  **Repository (저장소)**: 프로젝트 파일과 변경 이력이 저장되는 곳. (Local / Remote)
2.  **Commit (커밋)**: 변경 사항을 확정하여 저장하는 **하나의 버전(스냅샷)**.
3.  **Working Directory**: 내가 실제로 작업 중인 공간.
4.  **Staging Area**: 커밋하기 전, 저장할 파일들을 골라놓는 **대기 장소**.

### 🛠️ Git 초기 설정 & 기본 명령어 (Local)

**1. 설치 확인 및 유저 설정** (최초 1회 필수)
```bash
git --version
git config --global user.name "내이름"
git config --global user.email "내이메일@example.com"
```

**2. 프로젝트 시작**
```bash
git init
# 현재 폴더에 .git 폴더가 생성되며 버전 관리가 시작됨
```

**3. 상태 확인**
```bash
git status
# 어떤 파일이 변경되었는지, Staging 되었는지 확인
```

**4. 파일 준비 (Staging)**
```bash
git add 파일명
# 특정 파일만 스테이징
git add .
# 변경된 모든 파일을 스테이징
```

**5. 버전 저장 (Commit)**
* **방법 A (추천)**: `git commit -m "커밋 메시지"`
    * 간단하게 한 줄로 메시지를 남길 때 사용합니다.
* **방법 B (실수 조심!)**: `git commit`만 입력 후 엔터
    * Vim 같은 텍스트 편집기가 열립니다.
    * **당황하지 말고**: `i` 입력(작성 시작) -> 메시지 타이핑 -> `Esc` 키 -> `:wq` 입력 후 엔터(저장 및 종료).

**💡 커밋 메시지 규칙**:
* 동사로 시작하는 것을 권장 (Add, Fix, Update 등).
* 무엇을, 왜 바꿨는지 명확하게 작성.
