# 📘 [AI 헬스케어] Git & GitHub 핵심 요약 및 실무 가이드

## 1. 📚 PDF 교재 핵심 요약 (이론편)

### 1-1. 버전 관리(Version Control)란?
- **정의**: 파일의 변경 이력을 체계적으로 기록하는 것. (타임머신 기능)
- **목적**:
  - 실수 복구 ("최종_진짜최종" 파일 생성 방지)
  - 협업 시 충돌 해결 및 책임 명확화
- **Git vs GitHub**:
  - **Git**: 내 컴퓨터(로컬)에서 돌아가는 버전 관리 **엔진(도구)**.
  - **GitHub**: Git 저장소를 올려두는 **웹 서비스(클라우드)**. 협업의 중심.

### 1-2. 핵심 용어 정리
- **Repository (저장소)**: 프로젝트 파일과 변경 이력이 담긴 폴더 (.git).
- **Commit (커밋)**: 의미 있는 변경 사항을 기록하는 **하나의 점(버전)**.
- **Branch (브랜치)**: 원본을 건드리지 않고 독립적으로 작업하는 **가지(작업 공간)**.
- **Staging Area**: 커밋하기 전에 파일을 골라 담는 **장바구니**.

### 1-3. Git의 3가지 공간 (작업 흐름)
1. **Working Directory**: 내가 지금 작업 중인 공간 (수정 중).
2. **Staging Area**: `git add`로 올라간 공간 (커밋 대기).
3. **Repository**: `git commit`으로 기록된 공간 (영구 저장).

## 2. 💡 Q&A 심화 정리 (실전편)
*PDF에는 없지만 우리가 대화를 통해 배운 중요한 실무 디테일들입니다.*

### Q1. `git init` vs `git clone`?
- **`git init`**: 빈 땅에 내 집을 처음부터 지을 때. (새 프로젝트 시작)
- **`git clone`**: 이미 지어진 모델 하우스를 통째로 가져올 때. (기존 프로젝트 가져오기)
  - **특징**: `clone`을 하면 `init`, `remote 연결`, `pull`이 한 번에 완료된 상태가 됩니다.

### Q2. 터미널의 `$` 표시는 뭔가요?
- **정답**: **프롬프트(Prompt)**라고 하며, "명령어를 입력하세요"라는 신호입니다.
- **주의**: 코드를 복사할 때 `$`는 빼고 그 뒤의 명령어만 입력해야 합니다.
- **CLI 꿀팁**: `echo "내용" > 파일명` 명령어로 마우스 없이 파일을 뚝딱 만들 수 있습니다.

### Q3. 보안 파일(`secrets`) 관리는?
- **문제**: 환자 개인정보나 비밀번호가 깃허브에 올라가면 의료법 위반 및 해킹 위험.
- **해결**: `.gitignore` 파일에 숨길 파일명을 적어서 Git이 아예 무시하게 만듭니다.
- **주의**: 이미 올라간 파일은 `.gitignore`에 적어도 안 지워지니 `git rm --cached`로 빼줘야 합니다.

### Q4. `fetch`와 `pull`의 차이는?
- **`git fetch`**: 원격 저장소의 변경 사항을 **확인만** 하는 것. (내 코드는 안 바뀜)
- **`git pull`**: 변경 사항을 확인하고(`fetch`) 내 코드와 **합치는(`merge`)** 것.
- **헬스케어 실무**: 무작정 `pull`하기보다, `fetch`로 먼저 충돌 여부를 확인하는 것이 안전합니다.

### Q5. 충돌(Conflict)이 나면 어떡하나요?
- **원칙**: 충돌 해결의 주도권은 **'내 컴퓨터(로컬)'**에 있습니다.
- **해결법**:
  1. VS Code에서 충돌 파일 열기 (`<<<<`, `====` 확인)
  2. 둘 중 하나를 고르거나, 둘을 섞어서 코드 수정
  3. `git add` (해결 신고) -> `git commit` (마무리) -> `git push`

## 3. 🚀 헬스케어 협업 필수 워크플로우 (따라하기)
*브랜치 생성부터 깃허브 연동, 푸쉬, 풀까지의 완벽한 흐름입니다.*

### 상황: '환자 데이터 분석(feature/analysis)' 기능을 개발해야 함

#### STEP 1: 프로젝트 가져오기 & 작업 공간 만들기
```bash
# 1. 깃허브에 있는 프로젝트 내 컴퓨터로 가져오기
git clone [https://github.com/my-org/healthcare-project.git](https://github.com/my-org/healthcare-project.git)

# 2. 프로젝트 폴더로 이동
cd healthcare-project

# 3. 안전한 작업 공간(브랜치) 생성 및 이동
# (main 브랜치는 건드리지 않습니다!)
git checkout -b feature/analysis
```

#### STEP 2: 작업하고 기록하기 (Local)
```bash
# 4. 파이썬 코드 작업 (예: 결측치 처리 로직 작성)
echo "print('결측치 제거 완료')" > analysis.py

# 5. 작업물 장바구니에 담기
git add analysis.py

# 6. 작업 이력 박제하기 (커밋)
# 헬스케어는 재현성이 중요하므로 메시지를 구체적으로 씁니다.
git commit -m "feat: 환자 데이터 결측치 전처리 모듈 추가"
```

#### STEP 3: 깃허브에 올리기 (Push)
```bash
# 7. 내 브랜치를 원격 저장소(origin)로 쏘아 올리기
git push origin feature/analysis
```
*(이후 깃허브 웹사이트에서 `Pull Request` 버튼을 눌러 팀원에게 검사를 받습니다)*

#### STEP 4: 최신 버전 받아오기 (Pull)
*동료가 코드를 수정했거나, 다른 컴퓨터에서 작업할 때*
```bash
# 8. 메인 브랜치로 이동
git checkout main

# 9. 원격 저장소의 최신 내용 가져와서 합치기
git pull origin main

# 10. (선택) 내 작업 브랜치에도 최신 내용 반영하기
git checkout feature/analysis
git merge main
```
