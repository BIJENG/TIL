# 🐳 [TIL] Docker 완벽 가이드: 개념부터 FastAPI + MySQL 실전 구축까지

---

## 💡 1. Docker(도커)란 무엇이며, 왜 써야 하는가?

### 1.1. 도커의 탄생 배경 ("내 컴퓨터에선 되는데?")

개발을 하다 보면 "내 컴퓨터(로컬)에서는 잘 실행되는데, 서버에 올리거나 친구 컴퓨터에서는 에러가 나요!"라는 상황을 무조건 겪게 됩니다. 운영체제(Windows/Mac), 파이썬 버전, 설치된 라이브러리 등이 미세하게 다르기 때문입니다.
도커는 이 문제를 해결하기 위해 **'프로그램이 돌아가는 환경 자체를 통째로 포장해서 배포하자'**는 아이디어에서 출발했습니다.

### 1.2. 도커의 핵심 장점

1. **완벽한 격리 (결벽증 유지):** 내 윈도우(Host OS)에 MySQL이나 여러 버전의 파이썬을 직접 설치할 필요가 없습니다. 내 컴퓨터를 어지럽히지 않고, 도커라는 '가상의 무균실(컨테이너)' 안에서만 프로그램을 돌립니다.
2. **언제나 동일한 환경 보장:** `Dockerfile`이라는 설계도만 있으면, 전 세계 어떤 컴퓨터에서든 100% 똑같은 환경을 1분 만에 만들어낼 수 있습니다.
3. **효율적인 자원 관리:** 다 쓰면 컨테이너를 종료(`down`)해버리면 끝입니다. 예전에 깔아둔 DB가 윈도우 백그라운드에서 몰래 메모리를 갉아먹는 일이 발생하지 않습니다.

## 🏗️ 2. 도커의 3대 핵심 개념 (설계도 -> 빵틀 -> 빵)

도커가 실행되는 과정은 다음 3단계로 이루어집니다.

### ① Dockerfile (설계도)

우리가 만들 앱(FastAPI)을 실행하기 위해 **어떤 환경을 구축할지 적어둔 메모장**입니다.

- `FROM python:3.13-slim`: "파이썬 3.13 버전이 깔려있는 텅 빈 방을 하나 구해줘."
- `WORKDIR /app`: "그 방 안에서 앞으로 모든 작업은 `/app` 이라는 폴더에서 할게."
- `COPY . .`: "내 윈도우에 있는 파일들을 방 안의 `/app` 폴더로 전부 복사해 줘."
- `RUN pip install -r requirements.txt`: "복사한 파일 중 requirements.txt를 읽어서 필요한 라이브러리(FastAPI, MySQL 접속기 등)를 다 설치해 줘."
- `CMD ["uvicorn", "main:app", "--host", "0.0.0.0", "--port", "8000"]`: "이 방이 켜지면 무조건 이 명령어를 실행해서 서버를 돌려!"

### ② Image (빵틀 / 압축파일)

`Dockerfile`을 실행(`build`)하면 만들어지는 **변하지 않는 원본 템플릿**입니다. 게임으로 치면 '설치 CD'와 같습니다.

### ③ Container (구워진 빵 / 실행된 프로그램)

만들어진 Image를 실제로 **메모리에 올려서 실행시킨 상태**입니다. 하나의 Image(빵틀)로 여러 개의 Container(빵)를 똑같이 찍어낼 수 있습니다.

## 🎼 3. Docker Compose: 여러 컨테이너의 지휘자

현대적인 웹 서비스는 서버(FastAPI) 하나만 덩그러니 있지 않습니다. 데이터베이스(MySQL)도 필요하죠. 이 두 개의 컨테이너를 따로따로 켜고 연결하려면 너무 복잡합니다.
이를 한 번에 관리해 주는 설정 파일이 바로 **`docker-compose.yml`** 입니다.

```yaml
services:
  api: # 1번 방: FastAPI 서버
    build: . # 현재 폴더의 Dockerfile을 읽어서 방을 만들어라.
    ports:
      - "80:8000" # 내 윈도우의 80번 포트로 들어오면, 도커 안의 8000번 포트로 연결해줘.
    depends_on:
      - db # 단, db 방이 먼저 켜진 다음에 켜져야 해!

  db: # 2번 방: MySQL 데이터베이스
    image: mysql:8.0 # Dockerfile 없이, 공식 MySQL 8.0 빵틀을 바로 가져다 써라.
    environment:
      MYSQL_ROOT_PASSWORD: 1234
      MYSQL_DATABASE: oz
    ports:
      - "33061:3306" # 내 윈도우의 33061번 포트를, 도커 안의 3306 포트와 연결해줘.
```

## 🚀 4. 꼭 외워야 할 실전 명령어 가이드

터미널(PowerShell 등)에서 `docker-compose.yml`이 있는 폴더로 이동한 뒤 사용합니다.

- **`docker compose up -d`** : 도커를 실행합니다. (`-d`는 백그라운드 실행을 의미하며, 이걸 붙여야 터미널을 계속 다른 용도로 쓸 수 있습니다.)
- **`docker compose up --build -d`** : 파이썬 코드(`main.py`)나 `requirements.txt`를 **수정했을 때 반드시 사용**합니다. 기존에 구워둔 빵(캐시)을 버리고 새롭게 빌드하라는 뜻입니다.
- **`docker compose down`** : 켜져 있는 모든 컨테이너와 네트워크를 깔끔하게 끄고 삭제합니다. (초기화)
- **`docker ps`** : 현재 실행 중인 컨테이너 목록을 봅니다. 여기서 **`STATUS`가 `Up` 또는 `Running`이어야만 정상 작동 중**인 것입니다. (`Created`나 `Exited`면 에러가 났다는 뜻)
- **`docker compose logs -f`** : 백그라운드로 도는 도커 내부에서 무슨 일이 일어나는지, 에러가 없는지 실시간 로그를 확인합니다. (종료는 `Ctrl+C`)
- **`docker exec -it [컨테이너이름] bash`** : 실행 중인 도커 방(리눅스) 안으로 직접 걸어 들어갑니다. DB에 접속해서 쿼리를 날려볼 때 유용합니다.

## 💥 5. [핵심 트러블슈팅] 포트(Port) 충돌의 이해

가장 흔하게 겪는 **`Only one usage of each socket address is normally permitted`** 에러의 원리입니다.

- **포트(Port)란?** IP 주소가 '건물 주소'라면, 포트는 '건물 안의 호실 번호'입니다.
- **에러 발생 원인:** 윈도우(호스트)의 `33060` 호실을 쓰겠다고 도커가 요청했는데, 알고 보니 내가 예전에 윈도우에 직접 설치해 둔 MySQL(`mysqld.exe`)이 부팅할 때부터 이미 그 호실을 차지하고(Listening) 있었던 것입니다. (하나의 호실은 한 명만 쓸 수 있습니다.)
- **해결책 1 (우회하기):** `docker-compose.yml`에서 내 윈도우 쪽 포트를 비어있는 `33061` 등으로 바꾼다. (`33061:3306`)
- **해결책 2 (뿌리 뽑기):** 윈도우 작업 관리자 -> [서비스(services.msc)]에 들어가서, 윈도우용 MySQL 서비스의 시작 유형을 '자동'에서 '수동'으로 바꾸고 중지시킨다. (메모리 낭비 방지 및 포트 확보)

## 🔗 6. [매우 중요] 컨테이너 간의 네트워크 통신 (DB 연결)

개발자들이 도커를 처음 쓸 때 가장 많이 하는 실수입니다.

### ❌ 절대 하지 말아야 할 짓: `localhost` 사용

FastAPI 파이썬 코드(`main.py`)에서 DB에 접속할 때 주소를 `localhost:3306`이라고 적으면 **절대 접속이 안 됩니다.**

- **이유:** 도커 안에서 `localhost`는 **"FastAPI 컨테이너 자기 자신의 방"**을 의미합니다. 자기 방 안에서 DB를 찾으니 당연히 에러가 납니다.

### ⭕ 정답: 도커 컴포즈 서비스 이름 사용

도커 컴포즈로 묶인 컨테이너들은 거대한 하나의 내부 공유기(네트워크)를 나눠 씁니다. 이때 **`docker-compose.yml`에 적어둔 서비스 이름(`db`, `api`)이 곧 접속 주소(Domain)**가 됩니다.

- **파이썬 접속 코드 예시:** `mysql+pymysql://root:1234@db:3306/oz`
- **해석:** "비밀번호 1234를 들고, **`db`**라는 이름을 가진 컨테이너 방의 3306번 문을 열고 들어가서 `oz` 데이터베이스를 찾아라."

## ⌨️ 7. MySQL CLI(명령어 창) 다루기 팁

`docker exec -it docker-db-1 bash`로 들어가서 `mysql`을 실행했을 때 겪는 상황들입니다.

- **명령어를 쳤는데 실행이 안 되고 `->` 화살표가 나올 때:**
  - MySQL은 세미콜론(`;`)이 나올 때까지 문장이 안 끝났다고 생각하고 계속 입력을 기다립니다.
  - **탈출 방법:** `\c` (Clear)를 치고 엔터를 누르면 지금 치던 문장을 깔끔하게 취소하고 프롬프트 상태로 돌아옵니다.
- **강제 종료 (`Ctrl + C` 주의):**
  - 명령어를 취소하려고 무심코 `Ctrl + C`를 누르면, 명령어만 취소되는 게 아니라 MySQL 프로그램 자체가 강제 종료되어 쉘(bash) 밖으로 튕겨 나갑니다.
- **정상 종료:** `exit` 또는 `\q` 를 입력하고 엔터를 누릅니다.
