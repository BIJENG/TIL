# 📘 머신러닝 기초 정리

## 1. 라이브러리(Library) vs 프레임워크(Framework)

### 📚 라이브러리 (Library)
- **내가 필요할 때 꺼내 쓰는 도구 모음**
- “**You call the Library**” (개발자가 주도)
- 특정 기능을 제공하는 보조 도구
- 예: NumPy, Pandas, Matplotlib

### 🏗️ 프레임워크 (Framework)
- **구조와 흐름이 미리 정해져 있는 틀**
- “**The Framework calls you**” (프레임워크가 흐름을 주도, IoC)
- 개발자는 프레임워크가 만든 틀 안에서 코드를 작성
- 예: TensorFlow, PyTorch, Django

## 2. scikit-learn이란?

`scikit-learn`은 **머신러닝 알고리즘을 쉽게 사용할 수 있도록 만든 파이썬 머신러닝 라이브러리**입니다.

### 🧩 주요 특징
1. **다양한 머신러닝 알고리즘 제공**
   - 지도학습  
     - 분류: 로지스틱 회귀, SVM, 결정트리, 랜덤 포레스트, KNN  
     - 회귀: 선형 회귀, 라쏘, 릿지
   - 비지도학습  
     - 군집화: K-Means, DBSCAN  
     - 차원축소: PCA

2. **일관된 사용 방식 (API統一)**

4. 가중치(Weight)란?

모델이 입력값의 중요도를 학습하면서 업데이트하는 값

“이 특징이 결과에 얼마나 영향을 끼치는가?”를 수치로 표현

딥러닝에서는 수백만~수억 개의 가중치를 학습하여 패턴을 파악

5. 데이터셋: Train / Validation / Test
📘 데이터셋 구성

훈련 데이터 (Train)
모델 학습용 (교과서 + 문제집)

검증 데이터 (Validation)
모델 성능 체크 및 튜닝 (모의고사)

테스트 데이터 (Test)
최종 성능 평가 (실제 시험)

✔ 왜 섞이면 안 되는가? (매우 중요)

Train에 있던 데이터를 Test에 넣으면
모델이 이미 외운 값이라 정확한 평가가 불가능

이를 **데이터 누수(Data Leakage)**라고 한다

6. 데이터 분할 비율 예시

7 : 1.5 : 1.5

6 : 2 : 2

8 : 1 : 1
데이터 양과 문제에 따라 다르게 설정할 수 있음

7. 머신러닝 학습 유형
🎯 지도학습 (Supervised Learning)

정답이 있는 데이터로 학습

예: 강아지/고양이 라벨이 포함된 이미지

주요 문제 유형:

분류(Classification)

회귀(Regression)

🎯 비지도학습 (Unsupervised Learning)

정답(label)이 없는 데이터에서 패턴을 스스로 찾음

“이 사진들을 비슷한 것끼리 나눠봐”

주요 기법:

군집화(Clustering) → 대표: K-Means

차원 축소(PCA)

✔ 오늘의 핵심

지도학습은 정답이 있다 / 비지도학습은 정답이 없다

8. K-Means 군집화 (비지도학습)

K값(군집 개수)을 미리 정해줌

데이터가 K개의 그룹으로 나뉨

예: 고객을 3개 그룹으로 자동 분류

📌 오늘 배운 핵심 요약

지도학습 vs 비지도학습 구분

지도학습: 정답 O

비지도학습: 정답 X

Train / Valid / Test는 절대 데이터 중복 금지

중복되면 모델이 외워서 성능이 왜곡됨 → 데이터 누수
